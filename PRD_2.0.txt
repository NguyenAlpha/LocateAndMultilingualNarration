## 2) Tổng quan sản phẩm

### 2.1 Mô tả ngắn gọn
Hệ thống cung cấp trải nghiệm tham quan thông minh tại Phố Ẩm Thực: khách dùng app di động .NET MAUI để quét QR, chọn ngôn ngữ/giọng đọc, xem bản đồ gian hàng, nhận thuyết minh audio; đồng thời web admin phục vụ quản trị dữ liệu nghiệp vụ (gian hàng, nội dung, media, người dùng, gói dịch vụ).

### 2.2 Thành phần hệ thống
- **Mobile App (MAUI)**: cho khách tham quan (anonymous theo DeviceId)
- **Web Admin (ASP.NET Core MVC/Razor views)**: cho Admin, BusinessOwner
- **API Backend (ASP.NET Core Web API)**: nghiệp vụ, bảo mật, lưu trữ, tích hợp dịch vụ ngoài
- **Shared**: DTO dùng chung giữa Mobile/Web/API

### 2.3 Stack công nghệ (đối chiếu code hiện tại)
- **Mobile**
  - .NET MAUI `.NET 10` (Android/iOS/Windows/macOS)
  - CommunityToolkit.Maui, Mapsui.Maui, ZXing.Net.Maui, Plugin.Maui.Audio
  - SQLite (`sqlite-net-pcl`) cho offline cache
- **Backend API**
  - ASP.NET Core Web API `.NET 10`
  - EF Core 10 + SQL Server
  - JWT Bearer Auth + Refresh Token
  - Hosted service cho TTS background
- **Web Admin**
  - ASP.NET Core MVC (`AddControllersWithViews`) + Session
  - HttpClientFactory + AuthTokenHandler
- **Cloud/Integration**
  - Azure Speech (TTS)
  - Azure Blob Storage (audio/media)
  - Azure Translator

## 3) Vấn đề & Mục tiêu

### 3.1 Vấn đề hiện tại
1. Khách tham quan thiếu thông tin đa ngôn ngữ ngay tại điểm gian hàng.
2. Thuyết minh thủ công tốn nhân lực, không đồng nhất.
3. Cần vận hành ở môi trường mạng không ổn định (offline-friendly).
4. Cần quản trị nội dung tập trung cho nhiều gian hàng.

### 3.2 Mục tiêu sản phẩm
| Mục tiêu | KPI/Success Metrics |
|---|---|
| Tự động hóa trải nghiệm thuyết minh | Tỷ lệ vào flow thành công: Scan/Load → Map ≥ 95% |
| Đa ngôn ngữ thực dụng | Có danh sách language active + voice active, phát đúng preference |
| Ẩn danh nhưng cá nhân hóa | DevicePreference theo DeviceId lưu/khôi phục thành công |
| Trải nghiệm ổn định | App không crash khi API lỗi/mất mạng; có fallback local |
| Quản trị nội dung tập trung | Admin/BusinessOwner CRUD được business/stall/content |

## 5) Phạm vi sản phẩm

### 5.1 In Scope (theo trạng thái hiện tại)
- Mobile:
  - QR verify và session theo hạn dùng
  - Loading flow theo DeviceId + QR validity
  - Language/Voice selection, lưu DevicePreference
  - Map hiển thị stall + geofence + audio controls
  - Cache SQLite stall data + local audio path
  - Background sync định kỳ và theo connectivity
  - Profile cập nhật preference
- API:
  - Auth, Business, Stall, StallLocation, StallGeoFence
  - Language active/admin CRUD
  - TTS Voice active theo language
  - DevicePreference (GET/POST)
  - QR code quản trị + verify
  - Device location log batch
- Web:
  - Auth session, API client cho nhiều module quản trị
  - Nhiều controller admin/business đã có, mức hoàn thiện chưa đồng đều

### 5.2 Out of Scope (hiện tại)
- Social login
- Dashboard analytics hoàn chỉnh (BI/KPI sâu)
- Audit log đầy đủ
- Full auto geofence-trigger chuẩn production (còn tối ưu)
- Một số UX nâng cao theo TODO (featured stalls, nearest thực chiến hoàn thiện)

### 7.3 Quan hệ dữ liệu chính
- `Users` 1-N `RefreshTokens`
- `Users` N-N `Roles` qua `UserRoles`
- `Users` 1-1 `BusinessOwnerProfiles`, `EmployeeProfiles`, `VisitorProfiles`, `VisitorPreferences`
- `Businesses` 1-N `Stalls`; `Businesses` 1-N `SubscriptionOrders`
- `Stalls` 1-N `StallLocations`, `StallGeoFences`, `StallMedia`, `StallNarrationContents`
- `Languages` 1-N `TtsVoiceProfiles`, `StallNarrationContents`, `DevicePreferences`, `VisitorProfiles`, `VisitorPreferences`
- `StallNarrationContents` 1-N `NarrationAudios`
- `DevicePreferences` optional FK đến `TtsVoiceProfiles` (`VoiceId` nullable)

## 8.3 API Backend
- Đã có hệ thống endpoint phủ các module chính (Auth, Geo, Preference, QR, CRUD nghiệp vụ)
- Chuẩn hóa response qua `ApiResult<T>` cho đa số endpoint
- Có một số endpoint dạng raw DTO (ví dụ `api/stalls`) để tối ưu client cũ/tương thích

## 9) Luồng người dùng (User Flows)

### 9.1 Flow chính (actual trên Mobile hiện tại)
1. App mở `LoadingPage`
2. Kiểm tra `device_id`
3. Nếu chưa có QR hợp lệ → `ScanPage`
4. Quét QR thành công → `LanguagePage`
5. Chọn language + voice + confirm → lưu `DevicePreference`
6. Điều hướng `MapPage`
7. Map poll GPS, phát narration theo stall/geofence

### 9.2 Sequence Diagram (text)
```plaintext
Visitor -> LoadingPage : open app
LoadingPage -> QrService : IsAccessValid()
QrService -> API : POST /api/qrcodes/verify (khi scan)
LoadingPage -> DevicePreferenceApi : GET /api/device-preference/{deviceId}
Visitor -> LanguagePage : chọn language/voice
LanguagePage -> DevicePreferenceApi : POST /api/device-preference
LanguagePage -> MapPage : navigate
MapPage -> StallService : GET /api/geo/stalls?deviceId=...
StallService -> SQLite : cache/read
MapPage -> AudioGuideService : play/pause/stop
SyncBackgroundService -> SyncService : periodic sync
```

## 12) Yêu cầu phi chức năng

| Nhóm | Yêu cầu |
|---|---|
| Performance | Startup nhanh; map load ưu tiên local; API timeout 10s phía mobile client |
| Reliability | Không crash khi mất mạng; fallback local cache; sync retry theo chu kỳ |
| Security | JWT + refresh token hash; role-based authorization; phân quyền Admin/BusinessOwner |
| Offline-first | Stall data đọc từ SQLite trước; audio local path ưu tiên |
| Scalability | API stateless; cloud storage tách biệt; phân trang endpoints |
| Maintainability | MVVM + DI + Shared DTO; tách service rõ ràng; logging chi tiết |

## 13) Quyết định kỹ thuật quan trọng

1. **Device-based personalization** thay vì bắt buộc tài khoản mobile.
2. **Cache-first** (SQLite trước, API sau) để tối ưu UX.
3. **Mapsui/OSM** thay cho nền tảng map trả phí.
4. **Audio local-first** để giảm phụ thuộc mạng.
5. **API response envelope** `ApiResult<T>` cho xử lý lỗi nhất quán.
6. **Hosted background tasks** cho TTS/sync logic phía server/mobile.
